# Movie App

A Flutter application built using the MVVM architecture pattern to fetch and display movies from an API. Users can browse a list of movies and view detailed information about each movie on a separate page.

---

## Features

- Fetch and display movies from an API.
- Search functionality to filter movies by title.
- View detailed information about each movie on a dedicated detail page.
- Caching for improved performance.
- Utilizes modern Flutter development practices and tools.

---

## Tech Stack and Packages

### Core Framework:
- **Flutter**: The core SDK for building the application.

### State Management:
- **Riverpod**: For state management and dependency injection ([flutter_riverpod](https://pub.dev/packages/flutter_riverpod) ^2.6.1).
- **riverpod_annotation** and **riverpod_generator**: For simplifying Riverpod's boilerplate.

### Network and Data Handling:
- **http**: For making API requests ([http](https://pub.dev/packages/http) ^1.2.2).
- **json_annotation** and **json_serializable**: For JSON serialization/deserialization ([json_serializable](https://pub.dev/packages/json_serializable) ^6.8.0).

### UI and Utilities:
- **cached_network_image**: For efficient image loading and caching ([cached_network_image](https://pub.dev/packages/cached_network_image) ^3.4.1).
- **kartal**: Utility package for cleaner and more concise Flutter code ([kartal](https://pub.dev/packages/kartal) ^4.1.0).
- **logger**: For logging ([logger](https://pub.dev/packages/logger) ^2.5.0).

### Environment Variables:
- **envied** and **envied_generator**: For managing sensitive environment variables securely ([envied](https://pub.dev/packages/envied) ^1.0.1).

### Dev Tools:
- **build_runner**: To generate code for annotations ([build_runner](https://pub.dev/packages/build_runner) ^2.4.14).
- **very_good_analysis**: For linting and code analysis ([very_good_analysis](https://pub.dev/packages/very_good_analysis) ^7.0.0).
